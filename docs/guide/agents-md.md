# AGENTS.md

`AGENTS.md` 很适合拿来放那些你不想每次都重复提醒 AI 的规则。

对设计稿实现来说，很多偏差不是模型不会写，而是默认假设和你的仓库规则不一致。把这些规则前置到仓库里，通常比临时补一句话更稳。

Lanhu Context Mcp 的提示词会写得更加通用，这样也是为了交给开发者来编写 AGENTS.md 实现更强、更稳定的约束。

做个比喻，MCP tool 就像是一个组件，内部有自己的内置缺省提示词（变量）AGENTS.md 就是 MCP tool 的全局提示词（变量）

::: tip Claude
`Claude Code` 实际读取的是 `CLAUDE.md`，不是 `AGENTS.md`。如果你的仓库已经在用 `AGENTS.md` 给其他编码 Agent 提供规则，更贴近官方建议的做法是创建一份 `CLAUDE.md`，在文件顶部通过 `@AGENTS.md` 导入这份共享规则；如果有 Claude 专属约束，再继续写在导入语句下面。

官方文档：<https://code.claude.com/docs/en/memory#agents-md>
:::

## 处理忽略设计稿顶部有非业务性的顶部展示区域

设计师可能为了设计稿整体效果会把一些顶部的状态栏包含在设计稿中，我们可以利用提示词来去除（但是根本还是让设计师去除效果会更好）

```md
- 实现设计稿时，默认忽略所有非业务性的顶部展示区域；除非本次任务明确要求保留。非业务性的顶部展示区域包括但不限于：系统状态栏、设备外壳、展示用头部、返回按钮、页面标题、右上角胶囊按钮、机型边框。
```

## 优化设计稿转代码时外部容器高度经常是固定死的高度

蓝湖在转代码会死板的转具体的高度到 css 上，在移动端上，机型变幻莫测，肯定是想要高度自适应，而非固定死。测试发现**有些模型会自己判断处理**，如果有遇到模型不会判断处理，建议参考这个提示词

```md
- 最外层容器若有来自设计画布的固定高度，应去掉或者配置 min-h-screen 等，让内容撑开（除非固定高度属于设计意图，如固定头部、Banner 等）
```

## 优化设计稿转代码时蓝湖类名没有语义，后续不好维护

当你没有开启 tailwind 时，蓝湖默认的一些class 名称会相对没有那么语义化，比如 text-1、text-2，为了后续维护，建议参考这个提示词

```md
- 一定的语义化 class 类名，目前的类名可能是自增长的 text-1、text-2，请处理一定的语义化，方便后续维护
```

## 指定切图的下载位置

目前 MCP tool 有内置的提示词处理，让切图下载的位置在 `src/assets/{设计稿名称}/{icon-序号}.{ext}` 的格式。有让 AI 进行一定的语义化（但是可能和 AI 的能力有一定的关系），如果你想要调整具体的切图规则等，建议参考这个提示词

```md
- 切图下载的位置在 `src/assets/foo/bar/{设计稿名称}/`，icon 名称不能是 icon-1，icon-2，必须语义化
```

## AGENTS.md 参考模板

```md
# Repository Guidelines

## 设计稿实现硬约束

- 实现设计稿时，默认忽略所有非业务性的顶部展示区域；除非本次任务明确要求保留。非业务性的顶部展示区域包括但不限于：系统状态栏、设备外壳、展示用头部、返回按钮、页面标题、右上角胶囊按钮、机型边框
- 最外层容器若有来自设计画布的固定高度，应去掉或者配置 min-h-screen 等，让内容撑开（除非固定高度属于设计意图，如固定头部、Banner 等）

## pages 页面命名约定

- `pages` 目录下的 `.vue` 文件统一使用小写连接符（kebab-case）
```

:::tip
你可以在**设计稿实现硬约束**下面添加属于自己项目的硬约束
:::

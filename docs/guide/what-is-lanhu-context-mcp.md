# 什么是 Lanhu Context MCP？

## 解决什么问题？

`Lanhu Context MCP` 会把蓝湖设计稿整理成让 AI 生成代码还原页面的上下文。

它的核心目标不是直接产出最终业务代码，而是先交付一份稳定、可复用、便于下游模型继续实现的设计规格。

如果你熟悉 `Figma MCP`，可以把它先粗略理解成一种相近思路：不是让 AI 直接拿到代码，而是先把设计信息整理成让 AI 生成代码还原页面的上下文，再继续往项目实现推进。

`Lanhu Context MCP` 在这件事上有一点向 `Figma MCP` 的工作方式致敬，但重点放在蓝湖场景本身：

- 输入是蓝湖单张设计稿详情链接
- 输出是更偏实现侧的设计上下文，而不是直接替代工程代码
- 目标是把蓝湖稿更稳定地“喂给 AI”，为下游模型继续完成 Web、Android、iOS 等实现提供更可靠的上下文

## 输出内容

调用 `get_design_context` 后，通常会得到：

- **`HTML+CSS` 设计规格**：作为后续实现的主参考源，尽量保留设计稿里的结构、尺寸和样式细节
- **图片资源映射**：提供切图资源的原始地址、本地目标路径和下载命令，便于 AI 批量下载到项目目录，减少用户手动逐个处理资源
- **`Design Tokens`**：补充渐变、阴影、圆角等在主结构化规格之外仍有参考价值的视觉信息
- **设计图预览**：作为最终视觉对照，方便下游模型做结果校验
- **一段给下游模型使用的实现指引**：明确后续实现时的优先级、适配方式和约束条件

# Repository Guidelines

## 设计稿实现硬约束

- 实现设计稿时，默认忽略所有非业务性的顶部展示区域；除非本次任务明确要求保留。非业务性的顶部展示区域包括但不限于：系统状态栏、设备外壳、展示用头部、返回按钮、页面标题、右上角胶囊按钮、机型边框。
- 最外层容器若有来自设计画布的固定高度，应去掉或者配置 min-h-screen 等，让内容撑开（除非固定高度属于设计意图，如固定头部、Banner 等）
- 代码中要是没有提供具体的图片路径资源的话，就不要硬用 div 或者 svg 绘制了，让用户自己去处理
- 切图保存的路径要在 `src/assets/imgs/页面名称/xxx.xxx` 下

## pages 页面命名约定

- `pages` 目录下的 `.vue` 文件命名统一使用小写连接符（kebab-case）。
